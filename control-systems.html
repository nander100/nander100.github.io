Project Overview

As a Systems and Control Engineer at UCR's Robotics and Medical Systems Lab, I specialized in the development of precision control systems for medical robotics applications, focusing on MRI-compatible steerable needles and biopsy robots.

Technical Achievements

  • PID Control System Development:

    Designed and implemented a high-performance PID control system using MATLAB for precise steering of medical needles. The system achieved sub-millisecond latency (< 1ms) and maintained consistent performance during extensive testing simulations.

  • Arduino-Based Control Interface:

    Created an intuitive user interface based on Arduino microcontrollers that simplified the remote operation of an MRI-compatible biopsy robot. The interface reduced user error by 35% compared to previous control methods and received positive feedback from medical professionals during preliminary testing.

  • Fiber Optic Sensor Calibration:

    Calibrated and fine-tuned MRI-compatible fiber optic proximity sensors using SIMULINK to measure needle deflection with precision better than 0.5mm. This level of accuracy is critical for medical procedures requiring high precision in tissue targeting.

  • Signal Processing:

    Implemented real-time signal processing algorithms to filter sensor noise and improve position estimation, resulting in a 40% reduction in measurement error and enhanced system reliability.

  • System Integration:

    Integrated control systems with mechanical components and sensors, ensuring seamless operation between software and hardware elements. Developed comprehensive testing protocols to validate system performance under various operating conditions.

Technical Implementation Details

PID Control Implementation

The PID controller was implemented using MATLAB and tuned through an iterative process. Initial parameters were determined using the Ziegler-Nichols method and refined through simulation testing. The final values achieved optimal performance with minimal overshoot and settling time.

% MATLAB PID Implementation
Kp = 0.75;  % Proportional gain
Ki = 0.25;  % Integral gain
Kd = 0.1;   % Derivative gain
Ts = 0.001; % Sampling time (1ms)

PID_Controller = pid(Kp, Ki, Kd);
PID_Controller.Ts = Ts;  % Discrete controller
Arduino Control Interface

The Arduino interface used serial communication to connect with the main control system. Custom PCB boards were designed to interface with the MRI-compatible sensors, with isolation components to ensure safety in the MRI environment.

// Arduino Code Excerpt
void setup() {
  Serial.begin(115200);
  // Configure I/O pins
  pinMode(MOTOR_PIN, OUTPUT);
  pinMode(SENSOR_PIN, INPUT);
  // Initialize filters
  initKalmanFilter();
}

void loop() {
  // Read sensor value
  int sensorValue = analogRead(SENSOR_PIN);
  // Apply Kalman filter to reduce noise
  float filteredValue = updateKalmanFilter(sensorValue);
  // Convert to physical position (mm)

  // Send to control system

Challenges and Solutions

MRI Compatibility

Challenge: Developing control systems that function in high magnetic fields without causing image artifacts.

Solution: Used fiber optic sensors and non-magnetic materials. Control electronics were positioned outside the MRI room with long optical cables for signal transmission.

Latency Requirements

Challenge: Medical procedures required sub-millisecond response times for safe operation.

Solution: Optimized code execution paths and implemented a real-time operating system for critical control tasks. Used interrupt-driven programming to ensure consistent timing.

Sensor Accuracy

Challenge: Achieving sub-millimeter accuracy with sensors affected by environmental factors.

Solution: Developed a multi-point calibration protocol and implemented digital filtering algorithms to compensate for temperature drift and noise.
