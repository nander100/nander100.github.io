Organization Overview

As the Co-Founder and President of Highlander Combat Robotics at UC Riverside, I've led the development of a thriving robotics community focused on designing, building, and competing with combat robots. Our organization provides engineering students with hands-on experience in mechanical design, electrical systems, and programming while fostering teamwork and project management skills.

Technical Implementation Details

Robot Control Systems

Our robots feature Arduino-based control systems with custom firmware for precise motor control and telemetry. Teams implement different control strategies based on their robot design, from simple dual-joystick controls to more complex behavior-based systems.

// Arduino Motor Control Example
#include 

// Define motor control pins
#define LEFT_MOTOR_PIN 9
#define RIGHT_MOTOR_PIN 10
#define WEAPON_MOTOR_PIN 11
#define SAFETY_PIN 2

// Create servo objects for ESC control
Servo leftMotor;
Servo rightMotor;
Servo weaponMotor;

// Safety system variables
bool safetyEnabled = true;
unsigned long lastHeartbeat = 0;
const unsigned long HEARTBEAT_TIMEOUT = 500; // ms

void setup() {
  Serial.begin(115200);
  
  // Attach motor controllers
  leftMotor.attach(LEFT_MOTOR_PIN);
  rightMotor.attach(RIGHT_MOTOR_PIN);
  weaponMotor.attach(WEAPON_MOTOR_PIN);
  
  // Configure safety pin as input with pullup
  pinMode(SAFETY_PIN, INPUT_PULLUP);
  
  // Initialize motors to neutral
  leftMotor.write(90);
  rightMotor.write(90);
  weaponMotor.write(90);
  
  Serial.println("Combat robot control system initialized");
}
Safety System Design

The safety system incorporates multiple layers of protection, including physical kill switches, software timeouts, and remote emergency stops. Each robot's power system includes current monitoring and thermal protection to prevent battery failures.

// Safety System Implementation
void loop() {
  // Check physical safety switch
  if (digitalRead(SAFETY_PIN) == LOW) {
    emergencyStop("Physical kill switch activated");
    return;
  }
  
  // Check for communication heartbeat
  if (millis() - lastHeartbeat > HEARTBEAT_TIMEOUT) {
    emergencyStop("Communication timeout");
    return;
  }
  
  // Process incoming commands if safety checks pass
  if (Serial.available() > 0) {
    processCommand();
  }
}

Challenges and Solutions

Limited Budget

Challenge: Building combat robots with limited initial funding.

Solution: Developed partnerships with local machine shops for material donations, implemented a component recycling program, and created tiered robot designs that allowed teams to start with lower-cost platforms and upgrade incrementally.

Varied Member Experience

Challenge: Accommodating members with widely varying technical backgrounds and skill levels.

Solution: Created a mentorship program pairing experienced members with newcomers, developed progressive workshop series starting from fundamentals, and structured teams to include members with complementary skills.

University Regulations

Challenge: Navigating university safety requirements and space restrictions for combat robotics.

Solution: Collaborated with faculty advisors to develop comprehensive safety protocols, designed a modular arena with enhanced safety features, and established clear documentation for risk assessment and mitigation.
